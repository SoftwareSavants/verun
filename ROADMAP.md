# Roadmap

Where Verun is headed. Priorities shift as we learn - this is a living document, not a contract.

Track progress and discuss priorities in [Issues](https://github.com/SoftwareSavants/verun/issues) and [Discussions](https://github.com/SoftwareSavants/verun/discussions).

## Up next

- **Multi-agent support** - run Codex, Gemini CLI, and other coding agents alongside Claude Code
- **Subagent visualization** - see nested agent threads and their relationships in the chat
- **CLI parity** - full support for skills, slash commands, CLAUDE.md, AGENTS.md, and memory
- **Code review on diffs** - leave comments directly on changed lines
- **Idle LSP eviction** - stop language servers for inactive tasks to free memory, restart on focus

## Planned

- **Start command terminal + MCP integration** - surface errors from the project start command terminal and expose them via MCP so Claude can read logs, diagnose failures, and interact with the running process
- **Better plan mode** - richer plan review and editing experience
- **Customizable keybindings** - rebind any shortcut
- **Error recovery** - guided messages instead of raw technical errors
- **Editor picker** - "Open in..." submenu for VS Code, Cursor, Zed, and others
- **Scrollbar diagnostics** - show errors and warnings in the editor scrollbar gutter

## Exploring

Bigger bets we're thinking about but haven't committed to.

- Mobile companion app
- Voice mode
- Integrations (Linear, GitHub Issues, etc.)

## Accessibility

Ongoing work, not a single milestone.

- ARIA labels and focus management
- Keyboard-navigable UI
- Help panel and shortcut overlay
- Full accessibility audit
